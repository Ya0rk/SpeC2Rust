# RustAgent 文件生成计划机制

更新时间：2026-04-14

## 目标

给 `RustAgent` 增加可恢复的文件生成计划机制，避免以下问题：

- 每次重新运行都重复生成已经完成的文件
- 中断后无法续跑，只能从头开始
- 日志里看起来“计划过很多文件”，但磁盘上最终状态无法追踪

## 机制说明

`RustAgent` 现在会在 Rust 项目根目录下持久化一份计划文件：

- `.cgr_generation_plan.json`

其中包含：

- `project_structure`
- `implementation_plan`
- `planned_files`
- `files`
  - 每个文件的 `status`
  - 最近更新时间
  - `size`
  - `sha256`
  - 可选备注

## 文件状态

当前使用的状态包括：

- `pending`
- `in_progress`
- `completed`
- `failed`

## 执行规则

1. 先生成并解析项目结构
2. 再生成实现计划
3. 得到最终文件顺序后，初始化/更新 `.cgr_generation_plan.json`
4. 逐文件执行时：
   - 如果计划中状态是 `completed`，且磁盘上文件的 `size + sha256` 与记录一致，直接跳过
   - 如果磁盘上已有非空文件，即使计划没标完成，也会标记为完成并跳过
   - 只有 `pending/failed` 文件才重新生成
5. 每成功写入一个文件，就把状态更新为 `completed`
6. 每次跳过已完成文件时，会把该文件内容重新补回上下文，保证续跑时上下文连续
7. 如果某个文件在后续 `fmt/check/test` 中再次命中编译错误，修复器会把它在计划中的状态自动降级为 `failed`

## 额外调整

为了让“计划持久化”真正有效，`create_rust_project()` 也做了同步调整：

- 以前：如果项目目录已存在，会直接删除重建
- 现在：如果项目目录已存在，默认复用
- 只有目录存在但缺少 `Cargo.toml` 时，才重新初始化 cargo 项目

## 当前收益

这个机制能直接解决：

- 同一个输出目录反复运行时，已完成文件被重复生成
- 中途断流或手动中断后，无法从剩余文件继续
- 计划存在但没有执行状态记录
- 某个旧坏文件虽然曾被标记为 `completed`，但后续编译再次报错时，仍可在下一轮生成中重新进入待生成队列

## 当前限制

这套机制现在已经比“非空就跳过”稳很多，但仍有一个限制：

1. 旧版计划文件如果没有 `sha256/size`，第一次续跑时仍会把现有非空文件视为已完成，并在后续自动补齐元数据
2. 目前还没有“强制重生成”开关，手动要求某些文件重新生成时仍需要改状态或删除对应文件
